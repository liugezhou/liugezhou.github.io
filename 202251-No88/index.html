本周主题:年终小结

不觉间，2022年就要过去。
随着一年的时间在我们已经历的年岁长河中，所占据的跨度越来越短，一年也就弹指🫰之间。

这一年

这一年，
一月，迎接新生命；
二月，冬季奥林匹克运动会开幕、女足夺得第20届亚足联女子亚洲杯冠军；
三月，俄乌战争充斥头条、港股步入金融海啸；
四月，上海之声；
五月，2023年亚足联亚洲杯异地举办；
六月，中国神舟十四号载人飞船发射、唐山暴力攻击事件；
七月，河南村镇银行取款难、安倍晋三遇刺身亡；
八月，佩洛西访台激民愤；
九月，涉疫隔离转运ke车～侧翻事故；
十月，新班子走马、上任。
十一月，凡凡压火、卡塔尔世界杯拉开帷幕。
十二月，ChatGPT、新十条、通行码下线、羊羊羊。

这一年，每周小结伴我左右；
这一年，疫情肆虐，政策变化此起彼伏。
这一年，科技发展，实验室里热火朝天。
这一年，大脑清空，顺势而为。
这一年，明星娱乐不再占据头条。
这一年，没有新上映的中国电影。
这一年，皆在自保中闷声。

过去的一年，我都做了什么？留下些什么？翻看一下记录，总结一波，又是一个胶带。

每周小结

从今年三月份开始，以破局🔗为基点，开始了我新一轮的每周小结，截止到这一篇，无断更情况下，累计输出42篇。

从最初的随便写写，到开始慢慢成为一种周末陶冶情操的活动，这件事带给个人的好处暂时也说不清，只是能够坚持下来觉得还挺欣慰的。

做什么事情都有沉没成本，我想在写周报的同时，可能缺失了对技术某个方面的持续性深入研究、失去了周末把酒言欢的机会、失去了周末的更多可能性，可还好，看着一篇篇文章的产出，这种沉没成本可以承受。

关于每周小结的写作过程是这样的：

  1. Github上新建了一个私密仓库，所有的md文档的原始内容备份于此。
  2. 每周将看到的文章、工具、图片、鸡汤等内容、链接发送至微信对话框做素材收集。
  3. 每个周末打开公众号排版工具墨滴🔗，详细记录每一个素材，素材不够的情况下去刷推、看博客、Github等。
  4. 文章图床在Github上新建仓库，通过Picx🔗工具上传。
  5. 将写好的文章首先同步至博客，然后同步在公众号发布。

微信键盘

腾讯在 2021 年 1 月份就说要做键盘。
直到今年 9 月初才开始进行测试，直到最近几天，微信键盘 1.0.0 正式版才终于面世，目前已经可以在各大应用商店下载。

App Store 目前评分4.0(2398个评分)，低分评论主要为以下几点：
功能使用、不谈隐私、光标移动、兼容性不好等等。

关于之前的各种输入法存在的各种问题，在网上扒拉了一个图：

鉴于输入法被各种诟病，过度收集用户隐私，于是在下载微信键盘后，第一时间去看了一下用户隐私。
主要内容为：微信键盘分为隐私保护模式和拼写Plus模式。

默认的隐私保护模式下，输入的内容「不会上传至腾讯服务器，仅在本地设备存储」。
开启Plus模式，信息会被「上传至腾讯服务器」，经过「实时处理」，向用户返回对应的内容或服务，并且不会储存用户上传的信息。

当然了，一旦输入法将内容上传至云端，那么必定会经过后台数据分析，有被用于营销、推广、构建用户画像等的风险，因此非常介意"隐私信息泄露"的，还是不要开启Plus权限为好吧。

从隐私内容的文案来看，诚意满满，虽然腾讯的产品总被大家诟病，但在经过短短几分钟的体验后，算是最贴近iPhone原生体验的键盘，除了游标不习惯外，还是不错的，蹲后续吧。

Chrome 插件开发初体验

「需求」 每次打开 CSDN 的时候，都是由于未登录不能进行代码的拷贝。
然后每次我都在控制台输入： document.body.contentEditable = true。

于是，为了解决不登录 CSDN 也可以直接 copy，且不需要每次控制台输入代码的方式，使用 Chrome 插件的方式。 关于Chrome插件开发教程很多，这里记录一下我在解决不能代码拷贝情况下的过程：

新建一个项目 csdn-edit,进入项目新建文件：manifest.json

  {
    "manifest_version":2,
    "name": "Can Copy It",
    "version":"1.0",
    "description":"解决CSDN非登录情况下，不可以复制的尴尬",
    "icons":{
        "128":"img/logo.png",
        "48":"img/logo.png",
        "16":"img/logo.png"
    },
    "browser_action":{
        "default_icon":"img/logo.png"
    },
    "content_scripts": [
      {
        "matches": ["https://blog.csdn.net/*"],
        "js": ["script.js"]
      }
    ]
  }

logo添加忽略，新建script.js：

document.body.contentEditable = true

然后在Chrome扩展程序中，加载已解压的扩展程序即可。

页面关闭 favicon 请求🔗

让浏览器不发出 favicon 请求的方法：

<link rel="icon" href="data:,">

这行代码阻止了浏览器进行标准的favicon请求（通常是从favicon.ico中提取）。
通过添加这个元标签，可以告诉浏览器加载一个空的图像，而不运行另一个服务器请求。
本身节省了约400字节的带宽。

picture🔗

MDN文档中关于picture标签的介绍。

该标签通常包含零或多个source元素和img元素，来为不同的显示设备场景提图像版本。
Optimizing images with the HTML tag🔗 这篇文章就是用来介绍 picture 标签的使用。

  • 对于包含图片过多的网站，可以使用HTTP2服务器推送优化。
  • 几乎每个浏览器都支持 PNG、JPEG、GIF，但是新的 格式AVIF、WEBP、JPEG XL通常更有效、在不牺牲图片质量前提下体积更小。
  • 使用image magic🔗转换图像格式。
  • Safari目前不支持AVIF,WebP格式只在MacOS11+ 上工作。
  • 使用 picture标签的示例
<picture>
  <source srcset="/demo.avif" type="image/avif">
  <source srcset="/demo.jxl" type="image/jxl">
  <img src="/demo.jpeg" alt="Screenshot on iOS with selected text, showing a popup with options to make text Bold, Italic, or Underlined" width="375" height="153" class="border">
  </picture>
  • 还可以通过判断 User-Agent 头猜测浏览器使用哪种格式，从而获得一个很好的近似值。
